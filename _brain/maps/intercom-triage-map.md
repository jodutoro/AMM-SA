# Intercom T2 Triage — Map of Content

> Automated Intercom Tier-2 triage system. Claude classifies conversations, creates Linear tickets, links Intercom tracker tickets, and notifies ClickUp QA channels.

**Location:** `Work stuff/intercom-t2-triage/`
**Overview:** [[Work stuff/intercom-t2-triage/README|README]]
**Pipeline notes:** [[../memory/intercom-t2-triage-pipeline|Full pipeline + API gotchas]]

---

## Core Modules (`core/`)
| Module | Purpose |
|---|---|
| `ai_triage.py` | Claude-based triage logic |
| `clickup.py` | ClickUp QA channel routing map + notification builder |
| `devbot.py` | DevBot secondary validation |
| `intercom.py` | Intercom API — fetch convos, post notes, create+link tracker tickets |
| `linear.py` | Linear ticket creation |
| `state.py` | Processed conversation state (`data/processed.json`) |

## Entry Points
- `main.py` — FastAPI server (port 8001) — webhook receiver, triage result processor
- `run_triage_batch.py` — Batch runner: fetches T2 queue → triages via claude -p → posts results → sends ClickUp notifications

## Audit Scripts (separate from live pipeline)
- `audit_classifications.py` — One-off audit of SA - Type of Issue classifications (see [[../memory/intercom-classification-audit|notes]])
- `audit_sa_feature.py` — SA feature audit

## Data
- `data/processed.json` — Processed conversation state (dedup guard)
- Audit logs: `audit_log_YYYYMMDD_HHMMSS.csv` (auto-generated by audit scripts)

Context
Where the group is at
Members are actively building satellite/micro sites and starting to apply agentic workflows to real client work. The group is a mix of SEO-first and agency operators who are learning to think programmatically. A clear split is emerging between those who've pushed through the learning curve and are seeing dramatic results, and those still getting tripped up by tooling and setup.
Key Theme
File structure is the unlock
The single biggest takeaway from the session: having the correct root folder, AI rules markdown, and project-level memory files in place is what separates fluid, consistent builds from frustrating, broken ones. JD demoed his full setup live and it became an "aha moment" for multiple members.

Real blockers that came up — from tooling bugs to mindset shifts to client complexity.

Jonathan Giner · Justin Day · Bryan Fikes
Website Studio instability
Reported issues include: builds hanging during iteration, work lost without warning, sites disappearing after updates, and Warp pushes not arriving in Website Studio reliably. Jonathan Giner abandoned it entirely in favor of direct HTML/CSS builds via Claude Code.
Bryan Fikes · Multiple members
File structure confusion causing silent failures
Bryan's builds were routing to the wrong folder, making it look like Warp Agent was working when it wasn't. This is suspected to be widespread across the group. JD identified it as a root directory issue. The fix was demonstrated but not yet documented for everyone.
Group-wide
SEO-to-programmatic mindset shift
Most members come from SEO or agency backgrounds, not development. Thinking in terms of file systems, routing, and code structure is a real cognitive hurdle. Bryan literally printed a file structure diagram and put it on his wall as a reminder to "stop thinking like an SEO."
Bryan Fikes
Complex e-commerce database / plugin integration
A high-value client has a custom cart system with 68,000 SKUs, intentionally locked by the original developer. Connecting Claude to it via WordPress plugins is unsolved. The workaround being tested is building on top of the existing site rather than replacing it.
Clayton Joyner
Team skill gap for non-Webflow deployments
Clayton's team is not code-first. Even if Claude builds a clean site, deploying it outside Webflow — managing CDN, server config, etc. — makes his team nervous. He sees Webflow as essential infrastructure, not just preference.
Group-wide
No persistent memory between Claude sessions
Claude has no memory by default across sessions. Without deliberately set-up markdown files (AI rules, project context), every session starts cold. Most members had not yet fully implemented this, which was contributing to inconsistent results and repetitive prompting.

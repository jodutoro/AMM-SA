Clayton's Design Framework (Main Presentation)

  • Copy First, Design Second. Write all content before any design begins. This unlocks parallel processing and eliminates revision cycles. Nailing the homepage theme (colors, fonts, CTAs) cuts client revisions by 93%.
  • Design Style Taxonomy: Medical/Professional · Luxury/Red Carpet · Playful · High Fashion. Each category has preset component libraries so you're never starting from zero.
  • Approval gates at key stages: Theme → Homepage → Collection Pages → Core Pages. Human-in-the-loop only at decisions that matter.
  • Multi-agent ad creative pipeline: Research → Codify → Copy → Compliance Guard → Split Tests → Region Dialect Awareness. Agents extract process knowledge from ClickUp/Slack conversations and replicate it.

File Structure as the #1 Root Cause
Matt Bailey · Justin Day · Bryan Fikes · JD (screen share)
Setup & Config File Structure Workflows
▼

This was a diagnostic session. The group had homework from Manick: build satellite sites, make your main site bulletproof. The real discovery was that file structure problems are behind almost every Website Studio failure.

Member Updates

Bryan Fikes
Fully switched from Website Studio to building sites via Warp + Claude Code as raw HTML/CSS. Zero hiccups. Called it "life-changing" and "the next best thing to meeting Jesus." Now doing large multi-page sites this way.
Justin Day
Website Studio works well for location service landing pages — 3 prompts, done. Key: use it for simple/single pages, not multi-page builds. Unsure how to connect to root domain for SEO benefit.

JD's File Structure Rules (Screen Share)

  • Root directory is sacred. The one master folder you created at setup — stay there. Everything lives inside it.
  • For new projects: mkdir project-name → cd project-name → open Claude → start working.
  • For quick tasks: stay in root, trigger claude --dangerously-skip-permissions.
  • Always check your folder path in the terminal. File structure IS the reason Website Studio pushes fail.
  • SA is building vault-level architecture directly into the platform — infrastructure for your model with per-project vaults and linked markdown files.

"Stop thinking like an SEO. Think more like a scientist."

— Bryan Fikes (quoting the mental model shift)

Justin Day's Learning Loop System

  • After every session: feed the full session log to Claude. Ask: what did I do wrong? What could be better? Is file structure still intact?
  • Claude generates a full report. Add insights to memory so Claude learns and improves on the next session.
  • Result: Claude gets better at your workflow over time without you doing anything extra.

Other Key Points

  • Syncthing: Set it up before you start team deployments. Critical for file sync across machines and team members.
  • Google Drive + MCP: For galleries and before/after images, segment Google Drive the same way you segment the galleries. MCP reads and uploads directly — game-changing for image-heavy client sites.
  • SA MCP Playbooks: Ask Claude to "list your MCP workflows" — SA has handicrafted playbooks you can trigger directly from Claude Code.

Setup 1:1 Session

The AMM's first recorded 1:1 setup session. Justin Day onboarding from scratch — 3–4 year SA user, but never touched Claude Code. This session became the template for all member onboarding.

Research Agent + Obsidian + Claude Hook

What This Is

Three components turn your Obsidian vault into a passive intelligence engine:

ComponentWhat It DoesWhen
Insight Agent Reads vault context (voice notes, project docs, git history, team chat). Runs 4 analytical lenses through Claude to surface blind spots, design issues, and patterns. Scheduled / manual
Research Agent Picks top 3 topics from insights, queries Gemini for live data, synthesizes with Claude, writes HTML reports. After insight agent
Session Hook Claude Code SessionEnd hook. Captures git state (commits, branch, uncommitted changes) and writes a session log to the vault. Every Claude exit
The compounding effect: Each agent reads previous outputs. Insight reads session logs + past insights + research. Research reads insights. Over time, the system gets smarter about your blind spots.

Bootstrap Setup

Copy-paste this block to create the vault structure, install dependencies, and set up the env file. Replace VAULT with your vault path.

# Set your paths (edit these two lines)
VAULT="$HOME/Documents/My Vault"
ENV_FILE="$HOME/.env"

# Create vault directories
mkdir -p "$VAULT"/{Scripts,Insights,Research,Live,Wispr}

# Install Python dependencies
pip install anthropic python-dotenv google-generativeai markdown

# Create .env if it doesn't exist
cat > "$ENV_FILE" <<'EOF'
ANTHROPIC_API_KEY=sk-ant-...
GEMINI_API_KEY=AIza...
EOF

# Create Claude hooks directory
mkdir -p ~/.claude/hooks

# Verify prerequisites
python3 --version    # Need 3.10+
claude --version     # Need Claude Code CLI
git --version        # Required for session hook

Prerequisites checklist:

  • Obsidian — any recent version (no community plugins needed)
  • Claude Code CLI — installed and authenticated (claude --version)
  • Python 3.10+ with pip
  • git — the session hook reads git state
  • API keys — Anthropic (Claude) + Google Gemini, both billing-enabled

Configure the Intelligence Agents

Place 4 files in $VAULT/Scripts/. These are custom Python scripts you write — the sections below explain what each one does and how to configure it.

Scripts/ vault_report.py ← shared HTML renderer (both agents use this) insight-agent.py ← Stage 1: pattern detection research-agent.py ← Stage 2: deep investigation run-insight.sh ← wrapper for scheduled runs

vault_report.py — Shared Renderer

Converts markdown to dark-mode HTML reports. Both agents call render_report(title, content_md, ...) to produce standalone HTML files with metadata badges, navigation, and consistent styling. Uses the markdown Python library with fallback to regex conversion.

insight-agent.py — Stage 1

The core engine. Gathers context from 7 sources, budgets it to 120K chars, and runs analytical lenses:

LensSurfaces
blind-spotsAttention gaps, assumption risks, voice-vs-action disconnects
designAnti-patterns, complexity hotspots, API/data model issues
patternsCross-project themes, convergence opportunities, velocity
weeklyOne-page exec brief: scorecard, decisions needed, gaps

Context sources: project .planning/ states, architecture docs, voice notes (Wispr), git activity, ClickUp channels, previous insights, project summaries.

research-agent.py — Stage 2

Reads insights + voice notes, asks Claude to pick top 3 research topics (as JSON), queries Gemini for live data (SDK → CLI fallback → Claude fallback), synthesizes with Claude, writes one HTML report per topic.

run-insight.sh — Wrapper

Handles env loading for scheduled runs (launchd can't read .env). Extracts API keys using Python, syncs voice notes, runs the insight agent, and if the weekly lens ran, triggers the research agent + dashboard regeneration.

Configure paths at the top of each Python script:

# In insight-agent.py and research-agent.py:
VAULT = Path(__file__).parent.parent    # auto-detects from Scripts/ location
ENV_FILE = Path("/path/to/your/.env")   # edit this to your .env path

Symlink your projects into Live/

The insight agent reads PROJECT.md, STATE.md, ROADMAP.md, and CONCERNS.md from each symlinked directory. This gives it live visibility into all your active work.

cd "$VAULT/Live"
ln -s /path/to/project-a/.planning  project-a
ln -s /path/to/project-b/.planning  project-b
python3 "$VAULT/Scripts/insight-agent.py" --dry-run # should list context sizes, no API calls

Add the Claude Session Hook

This is the bridge: every time you close a Claude Code session, the hook auto-captures your git activity to the vault.

What it captures: repo name, branch, recent commits (12h), uncommitted changes, in-progress tasks.

A. Place the hook script at ~/.claude/hooks/obsidian-session-capture.py

The script reads a JSON payload from stdin (provided by Claude Code), runs git commands to gather state, and writes/updates a markdown file per repo in your vault.

Configure the output vault path at the top of the script:

OPS_VAULT = Path("/path/to/your/vault")
PROJECTS_DIR = OPS_VAULT / "projects"

B. Register it in ~/.claude/settings.json

Add this to the "hooks" object. If you already have hooks, merge the SessionEnd array:

{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python3 ~/.claude/hooks/obsidian-session-capture.py",
            "async": true
          }
        ]
      }
    ]
  }
}
How Claude Code hooks work: Claude Code fires shell commands on lifecycle events (SessionStart, SessionEnd, PostToolUse, etc.). The command receives hook context as JSON on stdin. Setting "async": true means it runs in the background and won't block Claude from exiting. The nested hooks array format supports multiple handlers per event with optional matchers.
Start and close a Claude session in any git repo. Check: ls "$VAULT/projects/" — should contain <repo-name>.md

Wire Up CLAUDE.md

~/.claude/CLAUDE.md is a markdown file that Claude Code reads at the start of every session. It acts as persistent instructions — telling Claude which tools to use, where credentials are, and what workflow to follow.

Why this matters for the intelligence loop:

  • CLAUDE.md tells Claude to use structured workflows (like GSD) that produce .planning/ artifacts — the exact files the insight agent reads.
  • It configures credential paths so all sessions use the same .env file.
  • It routes tasks to external tools (Gemini for live data, Codex for deep reasoning) — the same tools the research agent uses.

Minimum viable CLAUDE.md for this setup:

# My CLAUDE.md

## Credentials
All API keys live in `/path/to/.env`. Load via python-dotenv, never source.

## Git Rules
Commit messages: `feat:`/`fix:`/`docs:` prefix.
Never add Co-Authored-By trailers.

## Workflow
Use `.planning/` directories (PROJECT.md, STATE.md, ROADMAP.md)
for all structured work. These files are read by external
analysis agents to track project health.
The CLAUDE.md closes the loop. Without it, Claude sessions produce unstructured output. With it, every session generates the .planning/ artifacts that feed back into the insight agent. The more structured your sessions, the smarter the intelligence loop becomes.

Schedule Automation

Use macOS launchd to run the agents on a schedule.

Create ~/Library/LaunchAgents/com.vault.insight-agent.plist:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN"
  "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.vault.insight-agent</string>
  <key>ProgramArguments</key>
  <array>
    <string>/bin/bash</string>
    <string>/path/to/vault/Scripts/run-insight.sh</string>
    <string>--lens</string>
    <string>blind-spots</string>
  </array>
  <!-- Daily at 9 PM -->
  <key>StartCalendarInterval</key>
  <dict>
    <key>Hour</key><integer>21</integer>
    <key>Minute</key><integer>0</integer>
  </dict>
  <key>StandardOutPath</key>
  <string>/tmp/insight-agent-stdout.log</string>
  <key>StandardErrorPath</key>
  <string>/tmp/insight-agent-stderr.log</string>
</dict>
</plist>

Load it:

launchctl load ~/Library/LaunchAgents/com.vault.insight-agent.plist
Suggested schedule: Daily 9 PM for blind-spots (catches what you missed). Mon + Thu 8 AM for weekly (auto-triggers research agent). Create a second plist for the weekly run.
launchctl list | grep vault # should show your agent

Run & Verify

# Dry run (preview what will happen, no API calls)
python3 "$VAULT/Scripts/insight-agent.py" --dry-run

# Run a specific lens
bash "$VAULT/Scripts/run-insight.sh" --lens blind-spots

# Run research agent standalone
python3 "$VAULT/Scripts/research-agent.py"

# Research a specific topic
python3 "$VAULT/Scripts/research-agent.py" --topic "LLM cost optimization"

# Run all lenses (full intelligence sweep)
bash "$VAULT/Scripts/run-insight.sh" --lens all

Expected outputs:

WhatWhereFormat
Insight reports$VAULT/Insights/2026-04-13-blind-spots.htmlStandalone HTML
Research reports$VAULT/Research/2026-04-13-topic-slug.htmlStandalone HTML
Session logs$VAULT/projects/repo-name.mdMarkdown

HTML files render natively in Obsidian when clicked, and open directly in any browser. The dark-mode styling matches Obsidian's default theme.

Troubleshooting

ProblemCauseFix
Agent says "ANTHROPIC_API_KEY not set" The .env path in the script doesn't match your actual file Edit ENV_FILE at the top of insight-agent.py and research-agent.py
Gemini returns empty / "QuotaError" Free-tier quota exceeded or invalid key Check GEMINI_API_KEY is billing-enabled. Agent auto-falls back to Claude synthesis.
Session hook doesn't fire Hook not registered or async error Check ~/.claude/settings.json has the SessionEnd entry. Test manually: echo '{"cwd":"/tmp"}' | python3 ~/.claude/hooks/obsidian-session-capture.py
Session hook fires but no output Not in a git repo, or no recent activity Hook exits silently if git rev-parse fails or there are no commits/changes. Work in a git repo with at least one commit.
Launchd job not running Not loaded, or script path wrong launchctl list | grep vault. Check /tmp/insight-agent-stderr.log for errors.
.planning/ symlinks not reading Broken symlink or permission issue ls -la "$VAULT/Live/" — check symlinks resolve. On macOS, ensure Terminal has disk access.
Insight agent context too small No data sources connected yet Start with symlinked .planning/ dirs + a few voice notes. Context grows as you use the system.

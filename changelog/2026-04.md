# AMM Toolkit — What's New (April 2026)

> This is a curated digest for agency partners — filtered and explained in terms of what actually matters for your work.

## Claude Code — v2.1.111 (April 16, 2026)

**What changed:** New `xhigh` effort level for Claude Opus 4.7, tunable via `/effort`, `--effort`, and the model picker. Auto mode is now available for Max subscribers when using Opus 4.7.

**Why it matters for your agency:** `xhigh` sits between `high` and `max` — useful when you want more reasoning than `high` gives you but don't want to burn the full `max` token budget. For the audit + content generation workflows in this toolkit, `xhigh` is often the sweet spot: deep enough to catch nuanced SEO and compliance issues, cheap enough to run across a full client portfolio without hitting usage caps. Auto mode for Max subscribers means the model will adapt effort level per task automatically — let it handle the routing.

**Action needed:** Nothing required. If you are on the Max plan, try `/effort xhigh` on your next audit run and compare output quality. If you are running this toolkit's workflows programmatically, consider adding `--effort xhigh` to your default command for the audit and content-brief steps.

## Claude Code — v2.1.113 (April 17, 2026)

**What changed:** Several shipped together in this release:
- CLI now spawns a native Claude Code binary (via per-platform optional dependency) instead of the bundled JavaScript.
- New `sandbox.network.deniedDomains` setting blocks specific domains even when a broader `allowedDomains` wildcard would otherwise permit them.
- Fullscreen mode: `Shift+↑` / `Shift+↓` now scroll the viewport when extending a selection past the visible edge.
- `Ctrl+A` and `Ctrl+E` move to the start/end of the current logical line in multiline input (matches readline).
- Windows: `Ctrl+Backspace` now deletes the previous word.
- Long URLs in responses and bash output stay clickable when they wrap across lines (terminals supporting OSC 8 hyperlinks).
- `/loop` improvements: `Esc` now cancels pending wakeups, and wakeups display as "Claude resuming /loop wakeup" for clarity.

**Why it matters for your agency:** The native binary change is the headline — faster cold start and lower memory overhead during long audit or content-generation runs. `sandbox.network.deniedDomains` is the one to flag for client work: if you are running sandboxed research or code execution on behalf of a client, you can now block third-party domains (analytics, trackers, known bad actors) even inside a permissive allow-list. The readline-style Ctrl+A/Ctrl+E and Windows Ctrl+Backspace fixes make multiline prompt editing feel closer to a native shell. The `/loop` Esc cancel is the practical win for anyone running autonomous loops — you can now stop a pending wakeup without killing the whole session.

**Action needed:** Update to v2.1.113. If you run workflows inside a sandboxed config, review your `sandbox.network.allowedDomains` and add `deniedDomains` entries for anything you don't want leaking client-session traffic.
